C/C++小知识—unsigned int

unsigned int 没有负数
如果你很不巧写出了这样的代码:

那么你就会得到一个一个奇怪的结果 4294967295
并且这个结果也不是一个随机的值,而是一个固定值
这是为什么呢?

首先,理解一下有unsigned和没有的区别
在代码中如果没有unsigned,那么就将二进制编码的首位当作符号位,0代表正数,1代表负数
如果要表示-1的话,它的二进制编码大概长这样
1111 1111 1111 1111 1111 1111 1111 1111
再次强调一下,unsigned代表无符号,会将二进制编码的首位当作数据使用(此处”数据”表述不恰当,本意为不当符号使用,而是作为一个数)
所以硬要用它存数据,他直接计算那上面的一长串”1”,最终得到 4294967295
